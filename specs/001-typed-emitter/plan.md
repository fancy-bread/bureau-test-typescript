## Summary

Implement a generic `TypedEmitter<TEvents>` class that enforces event name and payload types
at compile time via mapped types and indexed access types. Provides `.on()`, `.once()`, `.off()`,
`.emit()`, and `.listenerCount()`. Zero runtime dependencies; all type safety is structural —
no runtime reflection. Existing `src/index.ts` placeholder is replaced.

## Phase 0: Research

**Decision**: Use a partial mapped record for the listener registry.
```typescript
type ListenerMap<TEvents extends EventMap> = {
  [K in keyof TEvents]?: Array<ListenerEntry<TEvents[K]>>
}
```
**Rationale**: Preserves per-key payload types without casting; satisfies strict mode.
**Alternatives considered**: `Map<string, Listener[]>` loses payload types; `Record<string, unknown[]>` requires `as` casts everywhere.

**Decision**: `ListenerEntry<T>` struct with a `once` boolean flag.
```typescript
type ListenerEntry<T> = { fn: Listener<T>; once: boolean }
```
**Rationale**: Avoids creating a new wrapper closure per `.once()` call; original reference survives for `.off()` matching.
**Alternatives considered**: Closure wrapping — simpler write path but `.off()` can never match the original reference, breaking US3/SC-4.

**Decision**: `.off()` matches on `entry.fn === listener` reference equality.
**Rationale**: Standard EventEmitter contract; predictable and trivially testable.

**Decision**: NodeNext import paths must end in `.js`.
**Rationale**: Required by `moduleResolution: NodeNext`; the compiler resolves `.js` to the corresponding `.ts` at build time.

## Phase 1: Design & Contracts

See [contracts/emitter-api.md](./contracts/emitter-api.md) for the full public API surface.

### Data Model

| Type | Definition | Notes |
|------|-----------|-------|
| `EventMap` | `Record<string, unknown>` | Constraint on the `TEvents` type parameter |
| `Listener<T>` | `(payload: T) => void` | Callback signature for a single event |
| `ListenerEntry<T>` | `{ fn: Listener<T>; once: boolean }` | Internal registry entry; not exported |
| `TypedEmitter<TEvents>` | Class | Owns `_listeners: ListenerMap<TEvents>` |

### Key TypeScript Constraints

- `TEvents extends EventMap` on the class declaration
- `K extends keyof TEvents` on each method
- `payload: TEvents[K]` derives the payload type from the key — no `any` needed
- `_listeners` entries are accessed with an existence check before use; no unguarded `!`
